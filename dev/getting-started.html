Getting Started

This tutorial walks through the core features of GeodeticOps.jl.

Creating Coordinates

The most common coordinate type is LLA (longitude, latitude, altitude):

# New York City (altitude defaults to 0)
nyc = LLA(-74.006, 40.7128)
LLA(lon=-74.006°, lat=40.7128°, alt=0.0m)
# Mount Everest summit
everest = LLA(86.925, 27.988, 8848.86)
LLA(lon=86.925°, lat=27.988°, alt=8848.86m)

All coordinate types are parametric. The type parameter promotes automatically:

LLA(1, 2, 3)        # Int
LLA(lon=1°, lat=2°, alt=3m)
LLA(1.0f0, 2.0f0)   # Float32
LLA(lon=1.0°, lat=2.0°, alt=0.0m)
LLA{Float32}(1, 2)   # Explicit type parameter
LLA(lon=1.0°, lat=2.0°, alt=0.0m)

Datums

A Datum defines the shape of the reference ellipsoid. GeodeticOps exports four common datums:

for d in [WGS84, GRS80, CLARKE1866, WGS72]
    println(d)
end
Datum("WGS 84", a=6.378137e6, 1/f=298.257223563)
Datum("GRS 80", a=6.378137e6, 1/f=298.257222101)
Datum("Clarke 1866", a=6.3782064e6, 1/f=294.978698214)
Datum("WGS 72", a=6.378135e6, 1/f=298.26)

All operations default to WGS84. You can override per-call with the datum keyword, or change the global default via DEFAULT_DATUM[]:

# Per-call override
distance(nyc, LLA(-0.1278, 51.5074); datum=GRS80)
5.585233578986238e6
# Change the global default
DEFAULT_DATUM[] = GRS80
DEFAULT_DATUM[]
Datum("GRS 80", a=6.378137e6, 1/f=298.257222101)
# Restore WGS84
DEFAULT_DATUM[] = WGS84
Datum("WGS 84", a=6.378137e6, 1/f=298.257223563)

Measuring Distances

Haversine gives a fast great-circle approximation:

london = LLA(-0.1278, 51.5074)
haversine(nyc, london)
5.570229848664454e6

Vincenty is more accurate, accounting for the ellipsoid:

distance(nyc, london)
5.585233578943206e6

The two methods agree to within ~0.5% for most distances.

Bearing

Compute the initial bearing (forward azimuth) from one point to another:

bearing(nyc, london)  # degrees clockwise from north
51.24122911946578

Computing Destinations

Given a starting point, bearing, and distance, find the destination:

# 1000 km due north from the equator
forward_geodesic(LLA(0.0, 0.0), 0.0, 1_000_000.0)
LLA(lon=0.0°, lat=9.042944436329899°, alt=0.0m)

Interpolation Along a Geodesic

Find the point at any fraction along a geodesic path:

tokyo = LLA(139.6917, 35.6895)

midpoint(nyc, tokyo)
LLA(lon=-153.64271957455668°, lat=69.72196141701565°, alt=0.0m)
# Quarter-way point
intermediate_point(nyc, tokyo, 0.25)
LLA(lon=-96.60006370071237°, lat=60.8407105106562°, alt=0.0m)

Coordinate Conversions

Convert between any supported coordinate types:

# LLA to ECEF (Earth-Centered, Earth-Fixed)
ecef = ECEF(nyc, WGS84)
ECEF(x=1.333998456149258e6, y=-4.654044842733645e6, z=4.1383002386952764e6)
# ECEF back to LLA
LLA(ecef, WGS84)
LLA(lon=-74.006°, lat=40.7128°, alt=0.0m)
# LLA to UTM
utm = UTM(nyc, WGS84)
UTM(zone=18N, easting=583959.372324113, northing=4.507350998397536e6)

Local coordinate frames (ENU, NED, AER) require a reference point:

# Displacement from NYC to a nearby point in East-North-Up
nearby = LLA(-73.996, 40.722, 50.0)
ENU(nearby, nyc, WGS84)
ENU(e=844.8835733062557, n=1021.7018510246394, u=49.86208985488406)

See Coordinates for the full list of types and conversion paths.

Polygon Area

Compute the area of a polygon on the ellipsoid:

# Approximate area of a 1 deg x 1 deg square at the equator
pts = [LLA(0, 0), LLA(1, 0), LLA(1, 1), LLA(0, 1)]
area_km2 = polygon_area(pts) / 1e6
println("$(round(area_km2, digits=0)) km^2")
12364.0 km^2

Bounding Boxes

Compute the bounding box of a set of points:

pts = [LLA(-74.0, 40.0), LLA(-73.0, 41.0), LLA(-75.0, 40.5)]
boundingbox(pts)
BoundingBox(lon=[-75.0, -73.0], lat=[40.0, 41.0])

Or find the bounding box of all points within a given distance:

destination_boundingbox(nyc, 50_000.0)  # 50 km radius
BoundingBox(lon=[-74.59771173143642, -73.41428826856358], lat=[40.26252811430199, 41.16303665323765])
