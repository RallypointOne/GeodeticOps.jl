GeodeticOps.jl

GeodeticOps.jl is a pure-Julia package for geodetic computations on Earth’s ellipsoid.

Features

  • 7 coordinate types with automatic conversions: LLA, LLARad, ECEF, ENU, NED, AER, UTM
  • Geodesic distances: Haversine (great-circle) and Vincenty (ellipsoidal)
  • Forward and inverse geodesic problems via Vincenty’s formulae
  • Geodesic interpolation: intermediate points and midpoints along geodesics
  • Polygon area on the ellipsoid using spherical excess
  • Bounding boxes: axis-aligned and distance-based
  • Multiple datums: WGS84 (default), GRS80, Clarke 1866, WGS72

Installation

using Pkg
Pkg.add("GeodeticOps")

Quick Example

using GeodeticOps

nyc = LLA(-74.006, 40.7128)
london = LLA(-0.1278, 51.5074)

d = distance(nyc, london)
println("NYC -> London: $(round(d / 1000, digits=1)) km")
println("Bearing:       $(round(bearing(nyc, london), digits=1))deg")
println("Midpoint:      $(midpoint(nyc, london))")
NYC -> London: 5585.2 km
Bearing:       51.2deg
Midpoint:      LLA(lon=-41.27790142901274°, lat=52.3894744727656°, alt=0.0m)

All operations default to WGS84. Pass datum=GRS80 (or any Datum) to override, or set DEFAULT_DATUM[] = GRS80 to change the global default.
